## Problem

Pull requests were being merged with no clear signal on which end-to-end tests covered the services they touched — leaving coverage gaps invisible until production.

## Approach

- **Led a 3-person team** to design, architect and ship in 24 hours.
- **Analysed PR diffs across repositories** and mapped impacted services to Jira-defined e2e test cases.
- **Surfaced intelligent coverage-gap recommendations** using LLM workflows over the GitHub and Jira APIs.

## Outcome

**Selected for the Jury Prize** from the full company-wide cohort. During development the tool already **surfaced several P1 and P2 issues** in the production application, which were reported to the owning teams.
