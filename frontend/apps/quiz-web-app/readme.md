# Quiz Web App

Standalone CSR React app for spaced-repetition flashcard study.

## Stack

- React 19 + Vite (strict CSR ‚Äî no SSR)
- TanStack Router (file-based routing)
- Zustand (state ‚Äî Phase 3+)
- Tailwind CSS 4 + shadcn/ui components
- Static quiz data from `@prj--personal-portfolio--v3/shared--quiz-export`

## Commands

```bash
# from monorepo root
pnpm --filter @prj--personal-portfolio--v3/frontend--quiz-web-app dev
pnpm --filter @prj--personal-portfolio--v3/frontend--quiz-web-app build
pnpm --filter @prj--personal-portfolio--v3/frontend--quiz-web-app typecheck

# refresh static JSON data (run after ingest)
pnpm --filter @prj--personal-portfolio--v3/shared--quiz-export start
```

Dev server: http://localhost:5180

## Data

Quiz content is served from `public/data/` (generated by `shared--quiz-export`):

- `posts.json` ‚Äî study-set index
- `tags.json` ‚Äî tag index
- `questions/<post_slug>.json` ‚Äî per-post questions
- `tags/<tag_slug>.json` ‚Äî per-tag questions
- `_all.json` ‚Äî full offline bundle

### Build pipeline

```
content repo (MDX/JSON)
  ‚Üí tools/content-sync          # clone content
  ‚Üí tools/mdx-ingest + json-ingest
  ‚Üí database/output/content.db  # SQLite (published rows)
  ‚Üí shared/quiz-export          # DB ‚Üí static JSON
  ‚Üí frontend/apps/quiz-web-app/public/data/*.json
  ‚Üí vite build                  # CSR bundle + PWA service worker
```

Regenerate the JSON after an ingest, then build:

```bash
pnpm --filter @prj--personal-portfolio--v3/shared--quiz-export start   # emit public/data
pnpm --filter @prj--personal-portfolio--v3/frontend--quiz-web-app build
```

## Offline / PWA

`vite-plugin-pwa` (autoUpdate) precaches the app shell plus `posts.json` /
`tags.json`. Per-post and per-tag question files are runtime-cached
stale-while-revalidate under the `quiz-data` cache, so any set you've opened
stays available offline. Icons live in `public/` (`pwa-192/512`, `apple-touch-icon`).
PWA assets are only emitted by `vite build`, not in dev.

## Routes

- `/` ‚Äî browse posts, add/remove study sets
- `/sets` ‚Äî your study sets with per-set counts
- `/sets/$postSlug` ‚Äî set detail; click any row to **preview** (stem, answer, explanation, SM-2 state); ignore/reset/remove
- `/sets/$postSlug/study` ‚Äî SM-2 study session for one set; `?cram=<slug>` for single-card cram
- `/study` ‚Äî study all due cards across every added set
- `/browse` ‚Äî global question browser (search, filter by format/difficulty/tag/state, paginated)
- `/tags` ‚Äî tags from your active sets
- `/tags/$tag` ‚Äî questions for a tag; study due or cram all
- `/tags/$tag/study` ‚Äî tag-scoped study session; `?cram=1` or `?cram=<slug>`
- `/stats` ‚Äî full progress: 30-day forecast grid, 1-year heatmap, ease distribution, reviews/day, retention rate, avg interval
- `/settings` ‚Äî theme, study order, SM-2 global config, progress backup/restore, clear all data

## Study session

Question stems, options, and explanations/answers are authored in **Markdown**
(headings, lists, fenced code, inline code, tables) and rendered via `CardRenderer`
(`marked` GFM + math tokenizer ‚Üí cloze substitution ‚Üí `DOMPurify` sanitize, in
`components/card/markdown.ts`). Option labels use inline Markdown. **Math** (`$‚Ä¶$`
inline, `$$‚Ä¶$$` block) renders with **KaTeX** and fenced code is highlighted with
**highlight.js** ‚Äî both are lazy-loaded/code-split (`lib/richText.ts`) and precached
for offline use. MDX is planned ‚Äî see
`_docs/02 plans/quiz-web-app-enhancements-plan.md`.

The study route builds a due queue (`selectStudyQueue`) scoped to the post,
then renders each card via `QuestionRenderer`, which branches on `answerFormat`:

- `free_text` ‚Üí reveal `explanation`, self-grade
- `multiple_choice` / `true_false` ‚Üí pick answer, auto-grade, then reveal
- `multiple_select` ‚Üí all-or-nothing auto-grade

Ratings (keys `1`‚Äì`4`, Space/Enter = Good) feed the SM-2 scheduler. Ignored
questions are excluded from every queue.
