# Quiz Web App

Standalone CSR React app for spaced-repetition flashcard study.

## Stack

- React 19 + Vite (strict CSR — no SSR)
- TanStack Router (file-based routing)
- Zustand (state — Phase 3+)
- Tailwind CSS 4 + shadcn/ui components
- Static quiz data from `@prj--personal-portfolio--v3/shared--quiz-export`

## Commands

```bash
# from monorepo root
pnpm --filter @prj--personal-portfolio--v3/frontend--quiz-web-app dev
pnpm --filter @prj--personal-portfolio--v3/frontend--quiz-web-app build
pnpm --filter @prj--personal-portfolio--v3/frontend--quiz-web-app typecheck

# refresh static JSON data (run after ingest)
pnpm --filter @prj--personal-portfolio--v3/shared--quiz-export start
```

Dev server: http://localhost:5180

## Data

Quiz content is served from `public/data/` (generated by `shared--quiz-export`):

- `posts.json` — study-set index
- `tags.json` — tag index
- `questions/<post_slug>.json` — per-post questions
- `tags/<tag_slug>.json` — per-tag questions
- `_all.json` — full offline bundle

### Build pipeline

```
content repo (MDX/JSON)
  → tools/content-sync          # clone content
  → tools/mdx-ingest + json-ingest
  → database/output/content.db  # SQLite (published rows)
  → shared/quiz-export          # DB → static JSON
  → frontend/apps/quiz-web-app/public/data/*.json
  → vite build                  # CSR bundle + PWA service worker
```

Regenerate the JSON after an ingest, then build:

```bash
pnpm --filter @prj--personal-portfolio--v3/shared--quiz-export start   # emit public/data
pnpm --filter @prj--personal-portfolio--v3/frontend--quiz-web-app build
```

## Offline / PWA

`vite-plugin-pwa` (autoUpdate) precaches the app shell plus `posts.json` /
`tags.json`. Per-post and per-tag question files are runtime-cached
stale-while-revalidate under the `quiz-data` cache, so any set you've opened
stays available offline. Icons live in `public/` (`pwa-192/512`, `apple-touch-icon`).
PWA assets are only emitted by `vite build`, not in dev.

## Routes

- `/` — browse posts, add/remove study sets
- `/sets` — your study sets with per-set counts
- `/sets/$postSlug` — set detail; 7-day forecast, SM-2 config glance, card table (ivl/ease), ignore/reset/remove
- `/sets/$postSlug/study` — SM-2 study session for one set
- `/study` — study all due cards across every added set
- `/stats` — full progress: 30-day forecast grid, 1-year heatmap, ease distribution, reviews/day, retention rate, avg interval
- `/settings` — theme, study order, SM-2 global config, progress backup/restore, clear all data

## Study session

Question stems, options, and explanations/answers are authored in **Markdown**
(headings, lists, fenced code, inline code, tables) and rendered via `CardRenderer`
(`marked` GFM + math tokenizer → cloze substitution → `DOMPurify` sanitize, in
`components/card/markdown.ts`). Option labels use inline Markdown. **Math** (`$…$`
inline, `$$…$$` block) renders with **KaTeX** and fenced code is highlighted with
**highlight.js** — both are lazy-loaded/code-split (`lib/richText.ts`) and precached
for offline use. MDX is planned — see
`_docs/02 plans/quiz-web-app-enhancements-plan.md`.

The study route builds a due queue (`selectStudyQueue`) scoped to the post,
then renders each card via `QuestionRenderer`, which branches on `answerFormat`:

- `free_text` → reveal `explanation`, self-grade
- `multiple_choice` / `true_false` → pick answer, auto-grade, then reveal
- `multiple_select` → all-or-nothing auto-grade

Ratings (keys `1`–`4`, Space/Enter = Good) feed the SM-2 scheduler. Ignored
questions are excluded from every queue.
